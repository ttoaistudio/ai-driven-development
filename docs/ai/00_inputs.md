# 00_inputs — 上流Inputs

## 目的
上流Inputを明文化し、以降の判断基準を固定する。

## Inputs（仮定）
- **プロダクト目的**: モノリス設計のベストプラクティス例を示すToDoアプリ
- **ユーザー**: 個人〜小規模チーム
- **トラフィック**: 低〜中（単一リージョン）
- **データ量**: タスク数 数百万規模まで
- **優先**: 迅速な反復 > 長期最適化
- **チーム**: 1〜3人
- **非目標**: マイクロサービス化

## 制約
- 単一アプリ
- 単一RDB
- 低オペレーションコスト
