# 40_hearing_inputs — 逆算のためのヒアリング項目

## 目的
要件定義を**逆算で成立**させるためのヒアリング項目を固定する。

## ヒアリング項目（Input）
| カテゴリ | 質問 | Output |
|---|---|---|
| 目的 | この実装で最短で達成したい価値は？ | 優先順位 |
| 制約 | チーム規模 / 期限 / 運用制約は？ | 設計制約 |
| 非目標 | 今回やらないことは？ | スコープ固定 |
| 技術 | 使う言語/フレームワークの制約は？ | 実装条件 |
| 品質 | どの品質特性を最優先する？ | テスト/設計方針 |

---

## Value（キャリア設計SaaS）ヒアリング結果
- 目的: 就活/転職/市場価値把握/キャリア形成のためのLP
- コア機能: 履歴書作成 / 職務経歴書作成 / 市場価値分析
- 体験: 質問に答えるだけで成果物が完成するフロー
- LPゴール: 登録（GoogleログインUI）
- LP構成: 一般的なセクション + デモ/イメージ重視
- 料金: 無料のみ
- デザイン: プレミアム感のあるブラック系（必要なら切替可能）
- 言語: 日本語のみ
